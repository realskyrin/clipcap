# Code-signing certificate

This directory holds the self-signed certificate used to sign **every** capcap
build ‚Äî local (`scripts/bundle.sh`) and CI (`.github/workflows/release.yml`).
Signing with one stable identity keeps the app's macOS TCC permission grants
(Screen Recording / Accessibility) intact across rebuilds and updates.

## Files (git-ignored ‚Äî never committed)

- `capcap-signing.p12` ‚Äî the certificate + private key. **Back this up.**
- `capcap-signing.p12.base64` ‚Äî base64 of the `.p12`, for the GitHub secret.

Default `.p12` password: `capcap` (set when generated by
`scripts/generate-signing-cert.sh`).

## First-time setup on a new machine

Import the cert so `codesign` can use the `capcap Self-Signed` identity:

```sh
security import scripts/signing/capcap-signing.p12 \
  -k ~/Library/Keychains/login.keychain-db -P capcap -T /usr/bin/codesign
```

## CI

Set these GitHub repo secrets (Settings ‚Üí Secrets ‚Üí Actions):

- `MACOS_CERTIFICATE` ‚Äî contents of `capcap-signing.p12.base64`
- `MACOS_CERTIFICATE_PWD` ‚Äî the `.p12` password (`capcap`)
- `MACOS_SIGNING_IDENTITY` ‚Äî `capcap Self-Signed`
- `KEYCHAIN_PASSWORD` ‚Äî any throwaway string

## Important

Losing this `.p12` (or regenerating it) gives the app a new identity, forcing
every user to re-authorize permissions once. Keep it backed up and reuse it.
