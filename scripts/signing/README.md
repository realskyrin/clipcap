# Code-signing certificate

This directory holds the self-signed certificate used to sign **every** clipcap
build ‚Äî local (`scripts/bundle.sh`) and CI (`.github/workflows/release.yml`).
clipcap uses an independent signing identity from the original app and does not
request screen or input-monitoring privacy grants.

## Files (git-ignored ‚Äî never committed)

- `clipcap-signing.p12` ‚Äî the certificate + private key. **Back this up.**
- `clipcap-signing.p12.base64` ‚Äî base64 of the `.p12`, for the GitHub secret.

Default `.p12` password: `clipcap` (set when generated by
`scripts/generate-signing-cert.sh`).

## First-time setup on a new machine

Import the cert so `codesign` can use the `clipcap Self-Signed` identity:

```sh
security import scripts/signing/clipcap-signing.p12 \
  -k ~/Library/Keychains/login.keychain-db -P clipcap -T /usr/bin/codesign
```

## CI

Set these GitHub repo secrets (Settings ‚Üí Secrets ‚Üí Actions):

- `MACOS_CERTIFICATE` ‚Äî contents of `clipcap-signing.p12.base64`
- `MACOS_CERTIFICATE_PWD` ‚Äî the `.p12` password (`clipcap`)
- `MACOS_SIGNING_IDENTITY` ‚Äî `clipcap Self-Signed`
- `KEYCHAIN_PASSWORD` ‚Äî any throwaway string

## Important

Losing this `.p12` (or regenerating it) gives the app a new signing identity.
Keep it backed up and reuse it for releases.
